## Recommended Sharing Sequence

```
Step 1: Share NDA template → collect signed NDA
Step 2: Share one-page overview → initial interest qualification
Step 3: Share evaluation scope → align on what is being reviewed
Step 4: Grant access (sandbox / repo) → reviewer begins evaluation
Step 5: Share audience-specific package (Technical / Academic / Practitioner)
Step 6: Collect feedback using template in evaluation scope document
Step 7: Debrief call
```

Any security findings discovered during evaluation must be reported via responsible disclosure to the designated security contact (see evaluation scope document Section 8). Do not file public GitHub issues for security vulnerabilities.
